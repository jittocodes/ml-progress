# Gaussian Mixture Models (GMM)

---

## Like a 5-year old

Imagine your class has tall kids and short kids. If you made a histogram of everyone's heights, you'd see two humps — one for the short group, one for the tall group. Each hump is a bell curve (Gaussian).

A Gaussian Mixture Model says: "this data looks like it came from several different bell curves mixed together." It figures out where each bell curve is, how wide it is, and how many people came from each one. Then it can tell you: "this kid probably came from the tall group."

---

## Technical Understanding

GMM is a **probabilistic clustering model**. It assumes data is generated by a mixture of K Gaussian (normal) distributions, each with its own mean and covariance.

**Key difference from K-Means:**
- K-Means: hard assignment — each point belongs to exactly one cluster
- GMM: **soft assignment** — each point has a **probability** of belonging to each cluster
- GMM can model **elliptical clusters** (not just spherical like K-Means)

**Applications in fintech:**
- Fraud detection: model normal spending as a Gaussian; anomalies have low probability under all components
- Customer lifetime value segmentation with uncertainty estimates
- Density estimation for risk models

## The Math

### Model Definition

Data comes from K Gaussian components:

```
P(x) = Σₖ πₖ * N(x | μₖ, Σₖ)

Where:
  πₖ = mixing weight (how common is component k), Σπₖ = 1
  μₖ = mean of component k
  Σₖ = covariance matrix of component k
  N(x|μ,Σ) = multivariate Gaussian distribution
```

### Gaussian PDF (1D)

```
N(x | μ, σ²) = (1/√(2πσ²)) * exp( -(x-μ)² / (2σ²) )
```

### EM Algorithm (Expectation-Maximization)

GMM is fit using EM — alternating between two steps:

**E-step (Expectation):** Compute responsibility — how much does each component "own" each point?

```
r_ik = P(component k | xᵢ) = πₖ N(xᵢ|μₖ,Σₖ) / Σⱼ πⱼ N(xᵢ|μⱼ,Σⱼ)
```

**M-step (Maximization):** Update parameters using the responsibilities:

```
πₖ  = (1/n) * Σᵢ r_ik
μₖ  = Σᵢ r_ik * xᵢ / Σᵢ r_ik
Σₖ  = Σᵢ r_ik * (xᵢ - μₖ)(xᵢ - μₖ)ᵀ / Σᵢ r_ik
```

Repeat until log-likelihood converges.

### Covariance Types

sklearn allows you to constrain the covariance structure:

| Covariance type | Shape | Parameters | Use when |
|---|---|---|---|
| `spherical` | Circles | 1 per cluster | Like K-Means |
| `diag` | Axis-aligned ellipses | d per cluster | Independent features |
| `tied` | Same ellipse for all | 1 matrix | Same spread, different centers |
| `full` | Any ellipse | d×d per cluster | Most flexible, most parameters |

## Pseudocode

```
FUNCTION gmm_fit(X, K, max_iters=100):
    # Initialize
    π = [1/K] * K                    # Equal mixing weights
    μ = K random points from X       # Random means
    Σ = [identity matrix] * K        # Spherical start

    FOR iteration in range(max_iters):
        # E-step: compute responsibilities
        R = zeros(n, K)
        FOR k in range(K):
            R[:, k] = π[k] * gaussian_pdf(X, μ[k], Σ[k])
        R = R / R.sum(axis=1, keepdims=True)  # normalize

        # M-step: update parameters
        Nk = R.sum(axis=0)  # effective number of points per component
        FOR k in range(K):
            π[k] = Nk[k] / n
            μ[k] = (R[:, k] @ X) / Nk[k]
            diff = X - μ[k]
            Σ[k] = (R[:, k] * diff.T) @ diff / Nk[k]

        # Check log-likelihood convergence
        log_likelihood = compute_log_likelihood(X, π, μ, Σ)
        IF converged: BREAK

    RETURN π, μ, Σ


FUNCTION predict_proba(X, π, μ, Σ):
    R = zeros(n, K)
    FOR k in range(K):
        R[:, k] = π[k] * gaussian_pdf(X, μ[k], Σ[k])
    RETURN R / R.sum(axis=1, keepdims=True)
```

## Python + Sklearn Implementation

```python
from sklearn.mixture import GaussianMixture
from sklearn.preprocessing import StandardScaler
import numpy as np

# --- Customer data with elliptical clusters ---
np.random.seed(42)
# Cluster 0: high balance, low txns (savers)
cov0 = [[1.0, 0.8], [0.8, 1.0]]  # correlated features
seg0 = np.random.multivariate_normal([1000, 5],   cov0, 200)
# Cluster 1: low balance, high txns (spenders)
cov1 = [[1.0, -0.5], [-0.5, 1.0]]
seg1 = np.random.multivariate_normal([200, 30], cov1, 200)
# Cluster 2: high balance, high txns (premium)
seg2 = np.random.multivariate_normal([5000, 50], [[1,0],[0,1]], 100)
X = np.vstack([seg0, seg1, seg2])

scaler = StandardScaler()
X_s = scaler.fit_transform(X)

# --- Fit GMM ---
gmm = GaussianMixture(
    n_components=3,
    covariance_type='full',  # most flexible
    n_init=5,
    random_state=42
)
gmm.fit(X_s)

labels = gmm.predict(X_s)
probs  = gmm.predict_proba(X_s)  # soft assignments

print(f"Converged: {gmm.converged_}")
print(f"Log-likelihood: {gmm.lower_bound_:.4f}")
print(f"BIC: {gmm.bic(X_s):.2f}")  # lower = better model
print(f"AIC: {gmm.aic(X_s):.2f}")

for k in range(3):
    n_pts = np.sum(labels == k)
    mean_orig = scaler.inverse_transform([gmm.means_[k]])[0]
    print(f"\nComponent {k} ({n_pts} points, weight={gmm.weights_[k]:.3f}):")
    print(f"  Mean balance: {mean_orig[0]:.0f}, Mean txns: {mean_orig[1]:.1f}")

# --- Anomaly detection ---
log_probs = gmm.score_samples(X_s)
threshold = np.percentile(log_probs, 5)
anomalies = X_s[log_probs < threshold]
print(f"\nAnomalies (bottom 5% log-probability): {len(anomalies)} points")

# --- Choose K using BIC ---
print("\n=== BIC for K selection ===")
for k in range(2, 7):
    g = GaussianMixture(n_components=k, covariance_type='full',
                        n_init=3, random_state=42)
    g.fit(X_s)
    print(f"K={k}: BIC={g.bic(X_s):.1f}")
```

## Key Interview Questions

**Q: GMM vs K-Means?**  
GMM gives **soft probabilities** and can model **elliptical clusters**. K-Means gives hard assignments and assumes spherical clusters. GMM is more flexible but slower.

**Q: What does the E-step compute?**  
The **responsibility** `r_ik` — the posterior probability that data point `i` came from component `k`. This is the "soft assignment."

**Q: How do you choose K for GMM?**  
Use **BIC** (Bayesian Information Criterion) or **AIC** — lower is better. Both penalize complexity (extra components), unlike raw log-likelihood which always improves with more components.

**Q: How can GMM detect anomalies?**  
Compute `score_samples(X)` — the log-probability of each point under the fitted model. Points with very low probability are anomalies (they don't fit any of the learned distributions).

---

## When to Use

✅ Data from overlapping, elliptical distributions  
✅ Need soft cluster memberships (probabilities)  
✅ Anomaly detection via probability scores  
✅ Density estimation  

❌ Very large datasets (EM is iterative, slower than K-Means)  
❌ Very high dimensions (covariance matrices become huge)  
❌ Clusters that are non-Gaussian (use DBSCAN)
